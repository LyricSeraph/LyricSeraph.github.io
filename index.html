备忘用

加密

参数：

Options Description
openssl OpenSSL command line tool.
enc Encoding with Ciphers.
-aes-256-cbc The encryption cipher to be used.
-salt Adds strength to the encryption.
-in Specifies the input file.
-out Specifies the output file.
-a Tells OpenSSL that the encrypted data is in Base64-ensode.
-pass Define the password.

Base64 encoding is a standard method for converting 8-bit binary information into a limited subset of ASCII characters.

It is needed for safe transport through e-mail systems, and other systems that are not 8-bit safe.

By default the encrypted file is in a binary format. If you are going to send it by email, IRC, etc. you have to save encrypted file in Base64-encode.

其中-pass的参数可以有：

  • env:somevar to get the password from an environment variable
  • file:somepathname to get the password from the first line of the file at location pathname
  • fd:number to get the password from the file descriptor number.
  • stdin to read from standard input

Java Keystore应用整理

Posted on 2018-04-21 Edited on 2019-12-17

Java Keystore

Java Keystore（JKS）这是Java的安全证书仓库，是对证书和密钥的管理。这种是基于Alias管理，一个Alias对应一套证书私钥，每个Alias可以单独设置密码。换言之，一个JKS文件可以包含多套证书私钥。

而管理这种仓库有个工具——keytool。

常用指令

指令功能 例子 备注说明
创建keystore和一个keypair keytool -genkey -alias ${Alias} -keyalg RSA -keystore keystore.jks -keysize ${密钥位数} -
查看keystore内容 keytool -list -v -keystore keystore.jks -
导入已有证书密钥（PKCS12） keytool -importkeystore -deststorepass {目标JKS的存储密码} -destkeypass ${导入后该密钥对的提取密码} -destkeystore keystore.jks -srckeystore import.p12 -srcstoretype PKCS12 -srcstorepass ${p12提取密码} -alias 1 keytool把p12文件当作一个jks文件处理，但是它不知道要导入哪个密钥对（即使只有1个），所以用-alias 1来指定被导入文件中的哪个密钥对，需要注意被导入的证书别名也为”1“
删除证书 keytool -delete -alias ${证书别名} -keystore keystore.jks -
修改keystore密码 keytool -storepasswd -new new_storepass -keystore keystore.jks -
修改证书对别名 keytool -changealias -keystore keystore.jks alias ${旧Alias} -destalias ${新Alias} -
导出证书 keytool -export -alias ${证书别名} -file certificate.crt -keystore keystore.jks -

SSL证书相关知识整理

Posted on 2018-04-20 Edited on 2019-12-17

X.509标准

X.509 标准的证书文件具有不同的编码格式，一般包括 PEM 和 DER 两种。。

编码 常见后缀 说明
PEM（Privacy Enhanced Mail） pem, crt 文本的格式，文件结构以 -----BEGIN XXX-----，并以 -----END XXX----- 结尾，中间 Body 内容为 Base64 编码过的数据
DER（Distinguished Encoding Rules） der, cer 二进制方式进行存储，文件结构无法直接预览

单纯一个公钥或者私钥同样有上述两种编码，但是实际使用中相当混乱，经常有串后缀名的情况，也常见key后缀但两种编码都有可能，所以最好根据内容判断。
其实一个证书事实上包含了公钥以及证书授权方的信息。

PKCS12

PKCS（Public Key Cryptography Standards） 是另一种标准，它定义了描述个人信息交换语法（Personal Information Exchange Syntax）的标准。PKCS12是PKCS系列的一个，可以用来将包含了公钥的 X.509 证书和证书对应的私钥以及其他相关信息打包，进行交换。即一个p12文件包含了一份x.509证书和一个私钥。并且，提取PKCS12密钥需要密码提取。

相关指令

OpenSSL命令，对常用的命令列个表

指令功能 例子 备注说明
生成私钥PEM openssl genrsa -out private.key ${密钥位数} 密钥位数要为2的整次幂，例如2048
根据私钥生成公钥 openssl rsa -in private.key -pubout -out public.key -
生成签名请求 openssl req -new -sha256 -key private.key -out sign_request.csr -
自签名证书 openssl x509 -req -days ${有效天数} -in sign_request.csr -signkey private.key -out certificate.crt 有效天数限制了证书的有效期
PEM证书转DER openssl x509 -in xxx.pem -out xxx.der -outform der 如果把x509改成rsa则是转换密钥格式
DER证书转PEN openssl x509 -in xxx.der -inform der -out xxx.pem -outform pem 同上
查看证书信息 openssl x509 -in certificate.crt -noout -text 同上，查看密钥信息
合并证书私钥为P12文件 ·openssl pkcs12 -export -out dest.p12 -in certificate.pem -inkey private.key 需要输入生成文件的提取密码
